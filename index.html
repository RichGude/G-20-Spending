Project Detail: Individual Project for DATS 6401: Analyses of Country Spending

In the rest of the pages of this site (accessed via the links or navigation bar above), the Google API and Charts functionality is used to illustrate and explore public services spending, namely education, healthcare, and military spending, across a swath of countries, for the purposes of visually identifying spending relations between each category and country GDP, per capita spending, or per capita growth.

Outcomes and Conclusions: if the purpose of this assignment was to gain familiarity with visual design techniques and practice HTML/CSS and Google API functions, the project was a complete success. Some takeaways for the author include the great functionality of BootStrap modules, from which this site was designed, and the easy intuitiveness of both the BootStrap and Google codes.

In contrast to this, of particular trouble for the author was the limited functionality of the Google API outside of sheets: effectively, using HTML code, there is little funcitonality for manipulating Google objects such as DataTables outside of extensive JavaScript handling. It begs the question, 'Why?' Why is the user able to transpose data tables or query multiple Googe Sheets to form a single data table when working from the Google Sheets page, but is not able to do the same with a simple, single, predefined JavaScript function (in the case of the querying multiple sheets, it appears this functionality is not present at all no matter the JavaScript skill)?

For this reason, the Google API has limited functionality, certainly over other tools such as visualizing Python or R libraries. Nevertheless, this was a great experience!
